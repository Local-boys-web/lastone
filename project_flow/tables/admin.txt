TABLE: ADMIN
Collection Name: admins

Description:
This collection stores all admin user data. Admins manage the entire platform, including job approvals, viewing all jobs, companies, users, and creating new admins.

Fields:
---------------------------------------------------------------------------
Field Name    | Type      | Description
---------------------------------------------------------------------------
_id           | ObjectId  | Unique ID (default generated by MongoDB)
email         | String    | Admin email for login (unique, required)
password      | String    | Hashed password (required)
name          | String    | Admin name (required)
role          | String    | Role, default: 'admin'
createdAt     | Date      | Timestamp, default: current date
---------------------------------------------------------------------------

Relationships:
- Admin can approve/reject jobs (reference to jobs collection)
- Admin can view all recruiters and users

Indexes:
- email (unique index for faster login queries)

Security Notes:
- Password must be hashed using bcrypt before storing
- Email validation required
- Admin cannot be created through public registration (only by existing admin)
