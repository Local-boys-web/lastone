TABLE: RECRUITERS
Collection Name: recruiters

Description:
This collection stores all recruiter/company data. Recruiters can post jobs, manage their job listings, and update their company profile.

Fields:
---------------------------------------------------------------------------
Field Name    | Type      | Description
---------------------------------------------------------------------------
_id           | ObjectId  | Unique ID (default generated by MongoDB)
name          | String    | Recruiter or company name (required)
email         | String    | Login email (unique, required)
password      | String    | Hashed password (required)
address       | String    | Company address (optional)
contactno     | String    | Direct contact number (stored as string)
createdAt     | Date      | Registration timestamp (default: current date)
---------------------------------------------------------------------------

Relationships:
- One recruiter can post multiple jobs (postedBy field in jobs collection)

Indexes:
- email (unique index for login)

Security Notes:
- Password must be hashed using bcrypt
- Email validation required
- Contact number stored as string to preserve formatting
